# Game Dev Resources

Game dev is a complex topic. People study for years to become a game dev, so don't think that this is a comprehensive guide.

# Unity

The industry standard as of writing. Most game dev positions expect competency with Unity. Getting started with Unity is pretty easy. Visit their website to select a plan and download the Engine.

Unity is free for students, and free for personal development as long as you make less than $100K per year on your games.

As far as learning how to use Unity, there are a plethora of tutorials on Youtube and accross the internet. Googling "Unity tutorial" alone yields about 76 million results.

Some of these may or may not be outdated though, so I tend to prefer Unity's own tutorials at learn.unity.com.

Unity uses C# for it's scripting language, so you're going to need some basic programming knowledge to get things working.

Unity has some visual scripting tools, but they are currently paid only.

# Unreal Engine

Often put head-to-head with Unity, Unreal is also industry standard. This is the engine they used to build Fortnite. You can get it at https://www.unrealengine.com/en-US/get-now.

While Unreal uses C/C++ as it's language of choice (which is quite advanced), you can also Unreal's blueprint system to code your game visually.

You may still have to dip into C++ if you need something that Blueprints can't do, or especially if you want to apply for a job that uses Unreal Engine.

# Godot

One of my favorite engines, Godot is completely free to use. When you download it you own everything. Not just your game code, but the engine code as well. Get it here https://godotengine.org/.

Godot has a visual scripting system as well as it's custom GDScript. They've recently even added C# support.

It's best to stick to one scripting system. Having multiple languages can make figuring out what's going on more difficult.
